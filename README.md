# Система управления доступом

## Уровни доступа
Система использует три фиксированных уровня доступа с возможностью расширения:
- **User** (уровень 1) - базовые права
- **Premium** (уровень 2) - расширенные права  
- **Admin** (уровень 3) - полные права

*Архитектура системы гибкая и позволяет добавлять новые уровни доступа при необходимости.*

## Правила доступа

### Доступ к данным:
- **User**: только данные с security_level = 1
- **Premium**: данные с security_level ≤ 2  
- **Admin**: все данные

### Административные функции:
- **Только Admin** может:
  - Получать данные о пользователях
  - Менять уровни доступа пользователей
  - Управлять пользователями
  - Создавать новые данные

## Эндпоинты API
- **`/auth/*`** - аутентификация и авторизация
- **`/data/*`** - работа с защищенными данными
- **`/admin/*`** - административные функции (только для Admin)

## Аутентификация и Авторизация

### Аутентификация:
- Реализована через JWT токены
- Пользователь получает токен при успешном входе
- Токен передается в cookies

### Авторизация:
- Каждый JWT токен содержит поля:
  - `user_id` - идентификатор пользователя
  - `access_level_id` - уровень доступа пользователя
- Уровень доступа из токена проверяется при каждом запросе
- Middleware автоматически проверяет права доступа к ресурсам
